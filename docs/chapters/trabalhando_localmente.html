Trabalhando localmente¶

Uma vez que um repositório é clonado, utilizaremos um conjunto restrito de comandos para trabalhar localmente (ou seja, sem enviar os dados para a nuvem, apenas no nosso computador). Na linha de comando do computador, e dentro da pasta do repositório, utilizaremos os comandos git status, git add, git restore, e git commit. Você pode ver uma explicação detalhada para cada um deles na Seção Comandos. No momento, é mais interessante saber que cada um dos comandos desempenha uma função diferente, e que a ordem em que eles são usados importa. Ou seja, utilizar o comando git commit (que cria um checkpoint) sem antes preparar os arquivos com git add não fará sentido.
